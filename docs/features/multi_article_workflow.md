# 「一詞多標，多篇並行」功能實作計劃 (Multi-Article Workflow Implementation Plan)

## 1. 概述
在目前的系統流程中，使用者進行一次關鍵字研究後，雖然會獲得 5 組 AI 建議標題，但目前僅能選擇其中一個來建立專案。若要針對轉化意圖不同的標題生成多篇文章，流程不夠流暢。本計劃旨在優化此工作流，允許使用者在單次研究後，針對多個感興趣的標題一鍵建立多個獨立專案，且各專案能無縫繼承已完成的研究數據。

## 2. 規格變動

### 2.1 後端變動 (API Layer)
*   **新增 `POST /api/projects/batch` 端點**：
    *   **Payload**: 
        ```json
        {
          "primary_keyword": "核心關鍵字",
          "country": "TW",
          "language": "zh-TW",
          "optimization_mode": "seo",
          "selected_titles": ["標題 A", "標題 B", "標題 C"],
          "keyword_cache_id": 123
        }
        ```
    *   **邏輯**：
        1. 從 `KeywordCache` (id=123) 讀取對應的研究數據 (`research_data`)。
        2. 針對 `selected_titles` 中的每個標題，建立一筆全新的 `Project` 記錄。
        3. 將該核心詞的研究數據、PAA、相關搜尋詞與所有建議標題同步寫入該專案。
        4. 回傳建立成功的專案 ID 列表。

### 2.2 前端變動 (User Interface)
*   **關鍵字研究結果頁面**：
    *   在 AI 建議標題列表左側加入 Checkbox（原有的「選用並建立」按鈕保留作單次快速入口）。
    *   下方新增「**為勾選標題建立專案 (並行開發文章)**」按鈕。
    *   點擊後執行批量建立 API，並在完成後導向專案列表或彈窗確認。

## 3. 效益與點數邏輯
*   **點數**：後續建立專案階段不額外扣點，充分利用已扣點的研究結果數據。
*   **數據一致性**：多個專案共用同一份研究基礎，但各專案可以有獨立的大綱與內容生成路徑。

## 4. 驗證計劃
*   [ ] 測試單次選取多個標題建立後，專案列表是否正確生成對應數量的項目。
*   [ ] 驗證每個專案是否都已正確填入研究回傳的內容（PAA, 相關搜尋等）。
*   [ ] 確認刪除其中一個專案不會影響其他專案或原始的研究快取。
