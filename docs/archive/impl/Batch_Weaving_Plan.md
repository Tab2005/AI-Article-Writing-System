# 批量編織自動化計劃 (Oracle Weaver Batch Processing Plan)

本計劃旨在為「因果矩陣」提供大規模、全自動的文章編織 (Weaving) 能力，並解決節點數量過多（如 500+）時的篩選與管理問題。

## 1. 核心選取與篩選機制 (Selection & Filtering)

針對大量生成的標題意圖，提供精確的選取手段：

- **維度過濾器 (Dimension Filters)**：
    - 使用者可透過「實體」、「動作」或「痛點」選單進行過濾。
    - 例如：僅顯示實體為「MetaMask」、痛點為「交易失敗」的所有組合。
- **多選勾選邏輯**：
    - **全選 (Select All)**：勾選當前視圖下的所有節點。
    - **條件選取**：支援「選取所有待編織 (Pending)」、「選取所有失敗 (Failed)」節點。
- **批量操作按鈕**：
    - 「⚡ 編織選中項 (Weave Selected)」：僅將手動勾選或篩選出的節點送入任務隊列。

## 2. 異步任務調度與並發控制 (Backend Operations)

- **後端隊列**：採用持久化任務隊列，確保即便瀏覽器關閉，生成任務仍能繼續。
- **頻率限制**：自動控管 API 請求頻率，支援並發但嚴格防止被封禁。
- **即時回傳**：透過 WebSocket 將每個節點的進度（Queued -> Weaving -> Done）即時推播至前端。

## 3. 面板進度追蹤 (Progress Management)

- **批量任務面板**：在介面右側或底部新增面板，顯示：
    - 目前總進度（如：已完成 45/500）。
    - 預估剩餘時間 (ETA)。
    - 「全部暫停 / 恢復」與「取消剩餘任務」功能。

## 4. 驗證

- 測試在擁有 500+ 節點的矩陣中，篩選出 50 個特定節點並啟動批量編織。
- 驗證「編織選中項」是否準確執行，且不會觸發未選取節點的生成。
