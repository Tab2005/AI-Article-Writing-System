## ⚠️ 操作方式變更（部署注意事項）

### 1. 雲端環境必須設定 `SECRET_KEY`

> [!CAUTION]
> 修復後，若 `SECRET_KEY` 未透過環境變數設定，**服務將無法啟動**。

**Zeabur 環境變數設定方式**：
1. 進入 Zeabur 控制台 → Backend 服務 → Environment Variables
2. 新增 `SECRET_KEY`，值請用以下指令產生：
   ```bash
   python -c "import secrets; print(secrets.token_hex(32))"
   ```
3. 若之前已設定過 `SECRET_KEY`，則無需額外操作

### 2. 500 錯誤回傳格式變更

修復前：
```json
{"success": false, "message": "伺服器內部錯誤", "detail": "具體錯誤訊息..."}
```

修復後：
```json
{"success": false, "message": "伺服器內部錯誤", "error_id": "a1b2c3d4"}
```

**前端影響**：若前端有解析 `detail` 欄位來顯示錯誤訊息，需改為顯示 `message` 或 `error_id`。

### 3. JWT Token 新增 `iat` 欄位

Token payload 新增了 `iat` (issued at) 時間戳，對現有前端邏輯無影響（向下相容）。

### 4. 本地開發環境

本地開發時，`.env` 檔必須包含有效的 `SECRET_KEY`（不能是預設值），否則無法啟動後端。可參考 `.env.example` 中的產生指令。
