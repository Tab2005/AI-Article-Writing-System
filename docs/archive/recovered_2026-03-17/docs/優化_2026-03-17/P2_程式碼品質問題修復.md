## 變更詳情

### P2-1 & P2-7：CMS Service 優化

**檔案**：`seonize-backend/app/services/cms_service.py`

- **統一匯入**：將 `or_` (SQLAlchemy)、`jwt`、`base64`、`re`、`markdown2` 等重複的局部匯入改為檔案頂部統一匯入，提升載入效率與可讀性。
- **連線管理**：移除 `GhostService` 與 `WordPressService` 初始化時建立的常駐 `httpx.AsyncClient`，改為在每個 API 呼叫方法內使用 `async with` 區塊，確保連線能及時釋放，防止雲端環境連線池洩漏。

### P2-4：統一時區 ISO 轉換

**檔案**：`seonize-backend/app/models/db_models.py`

- 新增 `_dt_to_iso(dt)` 內部工具函數，自動處理 SQLite 無時區 `datetime` 與 PostgreSQL 帶時區 `datetime` 的 ISO 格式轉換。
- 替換 `User`、`Project`、`KalpaNode`、`CMSConfig`、`CreditLog` 等模型中合計 19 處重複的 `.replace(tzinfo=timezone.utc).isoformat()` 邏輯。

### P2-5：快取過期邏輯重構

**檔案**：`seonize-backend/app/models/db_models.py`

- 引入 `ExpirableMixin` 類別，統一封裝 `is_expired` 屬性邏輯（包含時區自動校正比較）。
- `SerpCache`、`KeywordCache`、`CompetitiveCache` 改為繼承此 Mixin。
- 透過 `_default_expired` 屬性區分預設行為（SERP 預設為過期，其餘預設為不過期），大幅減少模型程式碼量。
