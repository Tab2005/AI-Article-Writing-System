## 變更詳情

### P4-2：統一日誌等級策略
- 清理了 `main.py` 中的啟動訊息，將原本的 `print()` 替換為 `logger.info()`。
- 確保所有生產環境的輸出都符合容器化最佳實踐（輸出至 StreamHandler）。

### P4-3：前端路由 Lazy Loading
- 透過 `React.lazy` 與 `Suspense` 對 `App.tsx` 進行了程式碼分割。
- 修正了原本 Named Export 頁面在使用 Lazy Import 時的型別相容性問題（使用 `.then(m => ({ default: m.Page }))` 模式）。
- **優點**：顯著降低了首屏 JavaScript 下載體積，改善使用者首次進入系統的載入體驗。

### P4-4：Pydantic Schema 獨立化 (Kalpa)
- 建立了 `app/schemas/kalpa.py` 並將原 API 層內嵌的模型（如 `KalpaSaveRequest`）抽離。
- API endpoint 現在使用匯入的 Schema 並指定了 `response_model`，提升了自動化文件的準確性與型別安全性。

### P4-5：完善 `.gitignore`
- 新增了 `uploads/` 目錄。
- 補充了 SQLite 的 `*.db-journal` 與專案日誌檔案排除規則。
