# Seonize 系統分階段優化計劃 (基於 2026-03-17 審查報告)

為了避免一次性大幅改動導致系統不穩定，本計劃將優化工作拆分為四個階段。**每個階段實作前，我都會重新提供當次詳細的檔案變更計劃並請求確認。**

---

## 階段一：安全性與核心穩定性 (P0 + P5 基礎) - [已完成]
> **目標**：修復致命安全性漏洞，並建立最基礎的自動化測試環境，確保後續改動不會造成崩潰。

### 主要變更內容
1. **安全性修復**：強制要求 `SECRET_KEY` 配置，修正 JWT 時區處理與 CORS 洩漏問題。
2. **穩定性補強**：修復 `kalpa_service.py` 中遺漏的 `math` 導入，避免批量編織失效。
3. **基礎測試**：建立 `tests/` 目錄並實作核心 API 的連線與認證測試。
4. **Bug 修復**：解決因果矩陣儲存後狀態不同步以及環境變數標籤顯示問題。

---

## 階段二：後端架構重構與效能 (P1 + P3) - [進行中]
> **目標**：優化資料庫交互與 AI 服務邏輯，解決因併發導致的潛在資料不一致。

### 主要變更內容
1. **資料庫關聯 (ORM)**：正式建立 `ForeignKey` 與 `relationship`（需搭配 Alembic 遷移），強化參照完整性。
2. **Session 管理**：統一使用 `get_db_context()`，避免手動關閉 `db` 導致的洩漏或衝突。
3. **AI 服務優化**：實作統一的 JSON 解析器與 AI 配置快取 TTL 機制。

---

## 階段三：前端組件模組化與型別安全 (P1 + P2)
> **目標**：拆分過於龐大的 `KalpaPage.tsx`，提升代碼可讀性與維護效率。

### 主要變更內容
1. **組件拆分**：將天道解析、矩陣配置、節點表格等拆分為獨立組件與 Hook。
2. **消除反模式**：移除 `window.refreshAuthUser` 全域掛載，改用標準 React Context。
3. **型別加固**：全面取代 API 層的 `any` 型別，定義精確的 Interface。

---

## 階段四：代碼清理與開發體驗優化 (P2 + P4)
> **目標**：移除冗餘代碼與過時檔案，優化全域 Log 與 CSS 結構。

### 主要變更內容
1. **代碼精煉**：使用 Mixin 處理過期邏輯，統一 datetime 格式化工具。
2. **清理環境**：移除 `audit_cms_fix.py` 等遺留除錯檔案。
3. **路由優化**：實作前端路由 Lazy Loading，提升首次載入速度。

---

3. **分段測試**：每完成一段優化，即執行對應的測試並回報結果。
